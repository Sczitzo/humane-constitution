## Introduction

This document walks through problems that have broken societies before. Each one is real. Each one has happened more than once, in places that never met. The record is there to read.

The chapters do not build on each other. You can start with any one of them. Each opens with a problem, follows it through real cases, and ends by pointing you to the part of this project that responds. Read in order, or skip to what you care about.

Three failure modes come up again and again. We give each one a plain name:

- **Survival-Trade Bind** — when staying alive depends on the market. If food, shelter, and medicine can only be bought, then losing your place in the market means losing your life.
- **Power-Wealth Convergence** — when wealth turns into power. Past a certain point, money buys laws, offices, and the rules themselves. The wealthy then use that power to protect their wealth.
- **Static-Advantage Loop** — when advantage passes down unchanged. Each generation inherits the last one's head start or its hole. Over time, that hardens into something that looks like caste.

One more thing shows up across the record, and we state it as an observation, not a belief. Across societies that never contacted each other, a shared moral-accountability framework — something that holds leaders answerable to a standard beyond their own interest — keeps appearing as a stabilizing factor. When it holds, institutions tend to correct toward serving people. When it erodes or gets captured, they drift toward the failures these chapters describe. This is a pattern in how institutions behave. It is not a claim that any religion is true or endorsed.

The chapters end with pointers into the rest of this project. They name the instrument or governance document that responds to each problem — Essential Access, Flow, Voice, Service Record, Shared Storehouse, and the governance files that track what is still unsolved. Follow those pointers when you want to see how the design answers the history.

A plain word before you start. This project does not claim to have solved these problems. It claims to have studied them carefully enough to try.
