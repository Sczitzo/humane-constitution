# Economic Spine Panel Review Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Produce a sober expert-panel review of the Commons Return, Universal Stake, Flow, anti-rent, and anti-capture architecture after the demurrage-to-hybrid rewrite.

**Architecture:** Use narrow expert lanes first, then synthesize only findings that survive cross-pressure. The output is a corpus-registered governance review plus small public-language corrections where the panel finds overclaiming.

**Tech Stack:** Markdown corpus, `scripts/export_corpus.py`, reader generated corpus, repository validation scripts, app build.

### Task 2: Synthesize Panel Review

**Files:**
- Create: `docs/governance/Economic_Spine_Anti_Capture_Coherence_Review.md`

- [ ] **Step 1: Write the review header and scope**

Include status, purpose, source documents, panel lanes, and claim boundary. The review must say it is not proof that the system works.

- [ ] **Step 2: Write the executive finding**

State whether the hybrid system should stay, be replaced, or be narrowed. Use direct language and evidence-status qualifiers.

- [ ] **Step 3: Write the scorecard**

Score fiscal adequacy, Flow coherence, legal-wrapper closure, ordinary-person protection, Christ-centered posture, implementation safety, and adversarial capture resistance.

- [ ] **Step 4: Write consensus findings**

Only include findings with concrete corpus consequences. Each finding needs: judgment, why it matters, affected docs, and action.

- [ ] **Step 5: Write dissent and unresolved issues**

List what the panel cannot prove from corpus text and what must go to simulation, legal review, pilot, or public testing.

- [ ] **Step 6: Write action queue**

Separate `Patch now`, `Evidence gate`, and `Do not claim yet`.
